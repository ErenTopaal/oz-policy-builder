how it works

From a transaction to an installed policy

Six deterministic stages. Each has a public command and a typed JSON output, nothing happens on-chain until the final, explicit install step.

01
record
Pull a transaction by hash, or simulate one, and decode every contract call, argument, asset movement and state change into a typed Recording.
02
synthesize
Derive the tightest context rule plus the minimum set of policies that permit exactly that flow. Bias toward least privilege, never a function or asset that wasn't observed.
03
codegen
Compose existing OZ primitives first; emit real, compilable Soroban policy contracts from audit-gated templates only where a primitive can't express the constraint.
04
simulate
Replay the recording as a permit case, then generate deny vectors, wrong function, wrong asset, over-limit amount, out-of-window timing, that the policy must reject.
05
export
Produce a wallet-signable install envelope as XDR. The tool builds and preflights it, it never submits anything on your behalf.
06
install
Sign with your wallet, install the context rule on your smart account, then read it back from chain and verify it matches what you reviewed.
No black box.
Every stage prints typed JSON you can read and verify.

architecture

Seven Rust crates, one TypeScript package

A virtual workspace over one core IR, with four interfaces, CLI, MCP server, agent skill, and wallet adapter.

rust
oz-policy-core
PolicySpec IR, decision tree, SEP-41 detection, Recording types.
rust
oz-policy-recorder
Soroban RPC client plus XDR decoder, hash or simulation to Recording.
rust
oz-policy-codegen
Askama templates, sandboxed compile, and five audit lint rules.
rust
oz-policy-simhost
In-process soroban-env-host harness with a proptest deny generator.
rust
oz-policy-installer
Install envelope builder with preflight and address registry, no submit.
rust
oz-policy-mcp
rmcp server, 5 tools, STDIO + HTTP, real on-chain readback.
rust
oz-policy-cli
Thin CLI over every crate: record / synthesize / codegen / simulate.
typescript
@oz-policy-builder/wallet-adapter
SEP-43 types, Freighter + passkey adapters, installPolicy + verifyInstall.
